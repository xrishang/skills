# ETF 资金流向分析 Skill

## 这个 skill 在做什么

给一个行业关键词(如"半导体")或"全市场 Top N"指令,抓取 ETF 实时资金流向,按主力净流入排序,从资金面/流动性/技术面/行业面四个维度分析,最后给出每只 ETF 的评分和排行。**与个股分析的根本区别**:ETF 不看公司财务(因为没有公司),核心是看**资金在往哪个行业/主题流入**。

四个维度:
1. **资金面** — 主力净流入/超大单/大单/中单/小单,谁在买
2. **流动性** — 规模(总市值)、成交额、换手率,能不能买得动
3. **技术面** — MA/MACD/RSI/KDJ,价格在什么位置
4. **行业面** — 跟踪指数、行业政策、行业景气度

**为什么是这四个维度**:ETF 的本质是"打包买一篮子股票",所以判断逻辑是"资金 + 行业 + 流动性 + 技术",不需要看公司财务。资金面告诉你聪明钱在往哪里走,流动性告诉你这个 ETF 适不适合你的体量,技术面告诉你短中期价格位置,行业面告诉你赛道好不好。

## 工作流总览

拿到指令后(行业关键词或 Top N),按以下顺序:

1. **识别模式** — 用户说"半导体 ETF"→industry 模式;用户说"今天 Top 20 ETF"→top-n 模式;用户给 ETF 代码→single 模式
2. **抓数据** — 跑 `scripts/fetch_etf_data.py`,抓全市场 ETF 实时行情 + 资金流向,按模式筛选并排序
3. **算指标** — 跑 `scripts/analyze_etf.py`,计算技术指标 + 资金面 + 流动性 + 行业估值,输出 `indicators.json`
4. **画图** — 跑 `scripts/plot_etf_charts.py`,生成资金流向排行图 + 价格趋势图 + 行业对比图
5. **写报告** — 读 `indicators.json`,按 `assets/etf_report_template.md` 写 Markdown 报告,对每只 Top N ETF 给出资金面/流动性/技术面/行业面评分 + 综合排名
6. **生成 PDF** — 跑 `scripts/generate_pdf.py`,把 Markdown + PNG 合并成 PDF(默认产出)
7. **综合评级** — 按加权分排序,给出"资金面 Top 3 ETF 推荐"+"需规避的资金净流出 ETF"

## 调用脚本的具体方式

所有脚本在 `scripts/` 下,从 skill 根目录运行。

### 1. 抓数据

```bash
# 全市场 Top 20 by 主力净流入
python scripts/fetch_etf_data.py --mode top-n --top 20 --out-dir ./output/top20

# 半导体行业 Top 10
python scripts/fetch_etf_data.py --mode industry --keyword 半导体 --top 10 --out-dir ./output/半导体

# 单只 ETF 详细分析
python scripts/fetch_etf_data.py --mode single --ticker 512480 --out-dir ./output/512480
```

输出:
- `etf_ranking.csv` — 排行榜(筛选后)
- `etf_spot_all.csv` — 全市场行情(备份)
- `price_<ticker>.csv` — Top 3 ETF 的历史日线
- `summary.json` — 汇总摘要,Claude 主要读这个

**数据源**:
- `fund_etf_spot_em()` — 1554 只 ETF,含主力净流入/成交额/流通市值/涨跌幅/换手率
- `fund_etf_fund_daily_em()` — ETF 类型分类(指数型-股票/海外/固收/其他)
- `fund_etf_hist_sina(symbol)` — 单只 ETF 历史日线(前复权,Sina 接口)

**行业匹配**:用内置关键词字典匹配 ETF 名称,如"半导体"匹配 [半导体,芯片,集成电路],"新能源"匹配 [新能源,光伏,锂电,电池,风电,储能,碳中和]。字典在 `fetch_etf_data.py` 的 `INDUSTRY_KEYWORDS` 里,可以扩展。

### 2. 算指标

```bash
python scripts/analyze_etf.py --out-dir ./output/半导体
```

输出 `indicators.json`,包含:
- 技术指标:MA5/10/20/60/120/250、MACD、RSI、KDJ、52 周高低、价格位置、趋势
- 资金面:主力净流入/净占比、超大单/大单/中单/小单、方向(流入/流出)、强度(强/中/弱)、主导力量
- 流动性:成交额、总市值、流通市值、换手率、份额、流动性等级(高/中/低)、规模等级(大/中/小)
- 行业面:行业、跟踪指数(从名称推断)
- 各维度评分(1-5 分) + 加权评分(资金面 35% + 流动性 25% + 技术面 20% + 行业面 20%)

### 3. 画图

```bash
python scripts/plot_etf_charts.py --out-dir ./output/半导体
```

输出:
- `charts/fund_flow_rank.png` — 主力净流入排行条形图(红=流出,绿=流入)
- `charts/price_trend.png` — Top 3 ETF 价格走势 + MA20/MA60
- `charts/industry_compare.png` — 行业内 ETF 规模 vs 成交额对比

### 4. 生成 PDF

完成 Markdown 报告 `report.md` 后,跑 `scripts/generate_pdf.py`(复用 stock-investment-analysis 的同名脚本):

```bash
python scripts/generate_pdf.py --md ./output/半导体/report.md --pdf "./output/半导体/半导体ETF资金流向分析报告.pdf"
```

**文件名规则**:`{主题}ETF资金流向分析报告.pdf`。主题取用户问的行业(如"半导体""医药""新能源")或"全市场"。用户问哪个行业的 ETF,PDF 名就对应那个行业。

## 四维度分析框架

每个维度打 1-5 分(5 分最好,3 分中性,1 分最差)。

### 维度 1:资金面 — "聪明钱在买什么"

**核心问题**:主力资金在流入还是流出,强度如何。

**关键指标**:
- 主力净流入-净额:正值=流入,负值=流出
- 主力净流入-净占比:净流入/成交额,衡量强度(>10% 强,<3% 弱)
- 超大单/大单/中单/小单净流入:看谁在主导(超大单主导=机构,小单主导=散户)

**打分锚点**:
- 5 分:主力净流入 >0,净占比 >10%,超大单主导
- 3 分:主力净流入 >0 但弱,或净流出但弱
- 1 分:主力强净流出(净占比 < -10%),超大单主导流出

**为什么资金面权重最高(35%)**:ETF 是"资金打包买赛道",资金流向是最直接的信号。主力净流入说明机构/聪明钱在加仓这个行业,反之是在减仓。

### 维度 2:流动性 — "买不买得动"

**核心问题**:规模够不够大,成交够不够活跃。

**关键指标**:
- 总市值:ETF 规模,>100 亿为大型,20-100 亿中型,<20 亿小型
- 成交额:当日成交金额,>10 亿为高流动性,2-10 亿中等,<2 亿偏低
- 换手率:成交额/流通市值,衡量活跃度

**打分锚点**:
- 5 分:总市值 >100 亿 + 成交额 >10 亿
- 3 分:中等规模 + 中等成交
- 2 分:规模 <20 亿 或 成交额 <2 亿(流动性差,买卖价差大)

**为什么流动性重要**:小规模 ETF 买卖价差大,大资金进出会冲击价格,跟踪误差也大。流动性差的不适合做交易标的。

### 维度 3:技术面 — "价格在什么位置"

**核心问题**:中期趋势、支撑压力位、超买超卖。

**关键指标**:
- 趋势:价格 vs MA60/MA250,多空排列
- MACD:DIF/DEA 金叉死叉
- RSI:>70 超买,<30 超卖
- KDJ:金叉死叉 + 超买超卖
- 52 周价格位置:当前价在过去 1 年的高低分位

**打分锚点**:
- 5 分:多头排列 + MACD 金叉 + 量价配合
- 3 分:震荡
- 2 分:空头排列

**技术面权重 20%**:ETF 是"指数跟踪",技术面更多反映市场情绪而非个股博弈,参考意义弱于个股。

### 维度 4:行业面 — "赛道好不好"

**核心问题**:ETF 跟踪的行业/主题是否在景气上行周期。

**关键问题**:
- 跟踪指数:沪深300/中证500/半导体/医药/新能源...
- 行业增速:行业增速 >15% 高增长,<5% 成熟
- 政策方向:国家支持(国产替代/新能源/AI+)还是限制(地产/教培)
- 产业链位置:上游/中游/下游

**打分锚点**:
- 5 分:行业增速 >15% + 政策强支持 + 公司是龙头指数
- 3 分:行业稳定 + 政策中性
- 1 分:行业拐头向下 + 政策打压

**行业面权重 20%**:由于行业 PE 分位等数据需要联网获取(当前环境 akshare 接口不稳定),行业面主要基于跟踪指数和政策方向定性判断。低信心度时建议用户自行核验行业估值。

## 评级规则

**权重分配**:
- 资金面 35%
- 流动性 25%
- 技术面 20%
- 行业面 20%

加权得分 = Σ(维度分 × 权重),满分 5 分。

**评级映射**(每只 ETF 独立评级):
- ≥4.0 且无维度 ≤2:**强烈推荐**(资金面+流动性+技术面共振)
- 3.5-4.0:**推荐**(资金面好但技术面待观察,或流动性中等)
- 2.5-3.5:**观望**(信号矛盾,如资金流入但价格位置高)
- <2.5 或资金面 =1:**回避**(资金在流出,不要逆势接飞刀)

**关键否决项**:
- 资金面 =1(主力强净流出)→ 最高"观望"
- 流动性 =2(规模太小)→ 最高"推荐"(小规模 ETF 不适合大资金)
- 技术面 =1(空头排列 + 放量下跌)→ 短期降级到"观望"

## 重要原则

1. **资金流向是核心**:ETF 分析的灵魂是看资金在往哪个行业流入,其他维度都是辅助
2. **不编数据**:akshare 接口偶尔抽风,抓不到的字段要明确写"数据缺失",不要瞎编
3. **数字优先**:每个判断先给数字,再给解读
4. **时效性提示**:Claude 训练数据有截止日期,行业政策、最新公告需提示用户自行核验
5. **不做投资建议**:报告末尾加免责声明

## 何时不用全套流程

- 用户只想看"今天资金流入 Top 10",直接跑 fetch + analyze,不写完整报告
- 用户只想看单只 ETF 的技术面,跑 fetch + analyze + plot 即可

## 何时升级

如果 akshare 某接口长期失败,看 `references/etf_apis.md` 找替代接口。如果用户想分析海外 ETF(美股 ETF)、商品 ETF(黄金/原油),当前 skill 不支持,问是否要扩展。
