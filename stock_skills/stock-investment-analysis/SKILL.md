# 个股投资价值分析 Skill

## 这个 skill 在做什么

给一只 A 股或港股,从六个维度判断"现在是否值得投资",最后给出一个明确评级。六个维度:

1. **公司财务** — 盈利能力、成长性、偿债能力、现金流质量
2. **估值** — 绝对估值(PE/PB/PEG/PS)与相对估值(历史分位、行业对比)
3. **技术面** — 趋势、支撑压力位、量价配合、超买超卖
4. **宏观经济** — 利率、汇率、CPI/PPI、政策周期对该股的影响
5. **行业景气度** — 行业增长、竞争格局、政策方向、产业链位置
6. **时事热点** — 近期新闻、业绩预告、突发事件对该股的影响

**为什么是这六个维度而不是更多或更少**:财务告诉你公司"过去好不好",估值告诉你"贵不贵",技术面告诉你"市场怎么看",宏观和行业告诉你"大环境顺风还是逆风",热点告诉你"短期有没有催化剂"。少一个就有盲区,多一个会稀释重点。每个维度都可能独立地否决一只股票(例如估值再便宜但行业拐头向下也难涨),所以评级是六维交叉验证而不是简单平均。

## 工作流总览

拿到股票代码后,按以下顺序做:

1. **识别市场和代码格式** — A 股 6 位代码(600519 / 000858 / 300750);港股 5 位代码 + .HK 后缀(00700.HK / 09988.HK)。如果用户只给公司名,先确认代码。
2. **调用脚本抓数据** — 跑 `scripts/fetch_data.py`,它会自动判断市场并抓取股价、财报、宏观、行业数据。脚本会输出一个 JSON 摘要 + 一份原始数据 CSV 目录。
3. **调用脚本算指标** — 跑 `scripts/compute_indicators.py`,输出技术指标(MA/MACD/RSI/KDJ)、财务指标(ROE/毛利率/净利率/资产负债率/净利增速)、估值指标(PE/PB/PEG/历史分位)。
4. **调用脚本画图** — 跑 `scripts/plot_charts.py`,生成股价趋势图、技术指标叠加图、财务指标趋势图,保存为 PNG。
5. **调用脚本做未来推演** — 跑 `scripts/project_future.py`,基于 indicators.json 和 summary.json 生成 `projections.json`,包含 3 场景财务预测(1-3 年营收/净利/利润率)和价格推演(6-12 个月目标价区间 + 技术面关键位)。可选生成 `projection_chart.png`。
6. **六维度分析 + 写 Markdown 报告** — 读脚本输出的 JSON 摘要(indicators + projections),按 `references/valuation_framework.md` 和 `references/rating_framework.md` 给每个维度打分(1-5 分,5 分最好)并写一段理由;按 `assets/report_template.md` 结构生成同名 Markdown 报告(如 `贵州茅台(600519)投资分析报告.md`)(引用 PNG 用 `![](charts/xxx.png)`)。报告包含"七、未来推演"章节。
7. **生成 HTML + PDF 报告(默认输出)** — 跑 `scripts/generate_pdf.py`,把 Markdown + PNG 合并成中文命名的 HTML + PDF(同名,均保留,如 `贵州茅台(600519)投资分析报告.html/.pdf`)。**用户最终拿到 PDF + HTML**,Markdown 是源文件。
8. **综合评级** — 按"评级规则"合成最终评级,给出 3 条核心理由 + 主要风险 + 建议跟踪指标(在 PDF 报告中体现)。

## 调用脚本的具体方式

所有脚本都在 `scripts/` 下,从 skill 根目录运行。每个脚本都接受 `--ticker` 参数(A 股 6 位代码或港股 5 位代码 + .HK),并支持 `--out-dir` 指定输出目录(默认 `./output/<ticker>/`)。

### 1. 抓数据

```bash
python scripts/fetch_data.py --ticker 600519 --out-dir ./output/600519
```

输出:
- `output/600519/price_daily.csv` — 日线行情(近 2 年)
- `output/600519/financials.csv` — 财报数据(近 5 年)
- `output/600519/macro.json` — 相关宏观指标
- `output/600519/industry_peers.csv` — 同行业公司列表
- `output/600519/summary.json` — 汇总摘要,Claude 主要读这个

脚本会自动重试并优雅降级:akshare 接口偶尔抽风,抓不到的字段会在 `summary.json` 里标 `null` 和原因,Claude 需要在报告中注明"该维度数据缺失"而不是瞎编。

### 2. 算指标

```bash
python scripts/compute_indicators.py --ticker 600519 --out-dir ./output/600519
```

输出 `output/600519/indicators.json`,包含三块:技术指标、财务指标、估值指标。所有指标都带"信号解读"字段(如 RSI=72 → 超买),Claude 可以直接引用但要自己再判断。

### 3. 画图

```bash
python scripts/plot_charts.py --ticker 600519 --out-dir ./output/600519
```

输出:
- `price_trend.png` — 股价走势 + 均线 + 成交量
- `technical_indicators.png` — MACD + RSI 副图
- `financial_trends.png` — ROE / 毛利率 / 净利率 / 营收增速 多年趋势
- `valuation_percentile.png` — PE / PB 历史分位

图表用 matplotlib 默认样式即可,不要花哨。中文标签需要用 SimHei 字体(脚本会自动设置)。

### 4. 未来推演

```bash
python scripts/project_future.py --ticker 600519 --out-dir ./output/600519
```

输出:
- `output/600519/projections.json` — 财务预测(3 场景,1-3 年)+ 价格推演(3 场景,6-12 个月)
- `output/600519/charts/projection_chart.png` — 历史 vs 预测营收/净利润对比图

脚本基于纯历史数据外推,不联网。使用 CAGR、利润率趋势斜率、估值倍数(PE/PS/PB)和技术面关键位(均线/52 周高低)生成三种情景(乐观/中性/悲观)。

**财务预测逻辑**:
- 营收:base = 历史 CAGR,bull = CAGR×1.3,bear = CAGR×0.7(下限 -30%)
- 利润率:base = 历史斜率,bull = 斜率×1.15+额外改善,bear = 斜率×0.5-恶化
- 净利润 = 预测营收 × 预测净利率(利润率有上下限 cap)
- 亏损公司:推算哪一年可能盈亏平衡

**价格推演逻辑**:
- 盈利 + PE 有效 → PE 法(预测 EPS × 目标 PE)
- 亏损或 PE 失效 → PS 法(目标 PS × 预测营收 / 股本)
- 股本无法推导 → 纯技术面(支撑/阻力位)
- 技术面关键位:支撑 = 52w低 + 均线(价格下方),阻力 = 52w高 + 均线(价格上方)

**信心度判断**:数据 ≥5 年 + 有 MA250 + 有 PE → 高;3 年或 2 项 → 中;不足 → 低。信心度低时,报告需强调"预测仅供参考,数据基础薄弱"。

### 5. 生成 PDF 报告(默认输出)

完成 Markdown 报告(如 `贵州茅台(600519)投资分析报告.md`)后,跑:

```bash
python scripts/generate_pdf.py --md "./output/600519/贵州茅台(600519)投资分析报告.md" --pdf "./output/600519/贵州茅台(600519)投资分析报告.pdf"
```

**文件名规则**:三种格式同名,`公司名(代码)投资分析报告` + `.md/.html/.pdf`。公司名取抓取数据里的全称(可去掉"股份有限公司/科技股份有限公司"等冗长后缀,保留核心字号);代码用 6 位 A 股代码或 `00700.HK` 形式港股代码。这样用户问哪只股票,最终三份报告名都对应那只股票,一目了然。

脚本用系统浏览器(Edge / Chrome)headless 模式渲染 HTML → PDF,中文字体由浏览器自动嵌入(微软雅黑),图片用 `file://` 绝对路径加载。生成后用 pypdf 裁掉浏览器默认加的页眉页脚(日期 + URL + 页码)。

**用户最终交付物是中文命名的 PDF + HTML(如 `贵州茅台(600519)投资分析报告.pdf/.html`)**,Markdown 是源文件。如果生成失败(找不到浏览器等),回退到交付 `.md`,并告诉用户"PDF 生成失败,给你 Markdown 版本;装 Edge 或 Chrome 后可生成 PDF"。

## 六维度分析框架

每个维度都要回答一个核心问题,并给 1-5 分(5 分最好,3 分中性,1 分最差)。打分要给理由,理由要引用具体数字,不要空话。

### 维度 1:公司财务 — "这家公司生意做得怎么样"

**核心问题**:盈利能力、成长性、偿债能力、现金流是否健康。

**关键指标**(参考 `references/valuation_framework.md`):
- 盈利能力:ROE(>15% 优秀)、毛利率(稳定或上升)、净利率(>10% 优秀)
- 成长性:营收增速、净利润增速(看 3 年 CAGR,不要只看 1 年)
- 偿债能力:资产负债率(行业差异大,要和同行比)、流动比率、速动比率
- 现金流:经营现金流 / 净利润(>1 为好,持续 <0.8 要警惕利润质量)

**打分锚点**:
- 5 分:ROE 持续 >20%,毛利率上升,净利增速 >20%,经营现金流/净利润 >1.2
- 3 分:ROE 10-15%,利润稳定,现金流和利润基本匹配
- 1 分:ROE <8% 且下滑,利润和现金流背离,负债率异常上升

**为什么这么打分**:ROE 是巴菲特最看重的指标,因为它衡量的是股东每一块钱投入的回报,直接反映生意好坏;现金流和利润背离是利润造假的最常见信号,所以要看两者匹配度。

### 维度 2:估值 — "现在买入贵不贵"

**核心问题**:当前价格相对于公司价值是便宜、合理还是昂贵。

**关键指标**:
- PE(TTM):A 股 15 倍以下偏低,30 倍以上偏高(行业差异大,要和同行比)
- PB:重资产行业看 PB,<1 倍破净
- PEG:PE / 净利增速,<1 偏便宜,>2 偏贵(成长股专用)
- PS:营收为负或周期股用 PS
- 历史分位:当前 PE/PB 在自己过去 5 年的分位(<20% 偏便宜,>80% 偏贵)
- 行业对比:和同行业可比公司比较

**打分锚点**:
- 5 分:PE 在历史 20% 分位以下,且低于行业平均,PEG <1
- 3 分:估值在历史中位附近,和行业相当
- 1 分:PE 在历史 80% 分位以上,PEG >2,显著高于同行

**周期股的陷阱**(重要):周期股 PE 低可能是周期顶点(利润最高时),PE 高可能是周期底部(利润最低时)。周期股要看 PB 和产品价格周期,不能简单看 PE。这也是为什么要交叉看行业维度。

### 维度 3:技术面 — "市场资金怎么看这只票"

**核心问题**:中期趋势、支撑压力位、量价配合、超买超卖。

**关键指标**:
- 趋势:股价 vs 60 日 / 120 日 / 250 日均线,多空排列
- MACD:DIF / DEA 金叉死叉,柱状图缩放
- RSI(14):>70 超买,<30 超卖
- KDJ:金叉死叉 + 超买超卖
- 量价:上涨放量 / 下跌缩量为健康,反之警惕
- 支撑压力位:近 1 年高 / 低点,整数关口,跳空缺口

**打分锚点**:
- 5 分:站上所有均线且多头排列,MACD 金叉,量价配合,RSI 中性
- 3 分:区间震荡,方向不明
- 1 分:跌破 250 日均线,MACD 死叉且柱状图放大,放量下跌

**为什么技术面要进六维**:技术面反映"市场对公司未来的预期已经定价了多少"。如果基本面好但技术面差,说明市场不认可你的判断,要么你错了要么早了,无论如何都不该立刻重仓。

### 维度 4:宏观经济 — "大环境顺风还是逆风"

**核心问题**:利率、汇率、通胀、政策周期对该股是利好还是利空。

**关键映射**:
- 利率下行 → 利好高估值成长股、地产、利率敏感行业
- 利率上行 → 利好银行保险,压制高估值股
- 人民币贬值 → 利好出口型制造业、纺织家电;利空航空、造纸
- CPI 上行 → 利好消费、农业;PPI 上行利好资源股
- 政策方向(财政刺激 / 行业监管 / 产业政策)

**打分锚点**:
- 5 分:宏观环境明确利好该股所在行业,且趋势延续
- 3 分:宏观中性或影响有限
- 1 分:宏观明确逆风(如地产链在去杠杆周期)

### 维度 5:行业景气度 — "赛道好不好"

**核心问题**:行业增长、竞争格局、政策方向、产业链位置。

**关键问题**:
- 行业增速:GAGR >15% 高增长,5-15% 稳定,<5% 成熟或衰退
- 竞争格局:CR3 / CR5 集中度,龙头 vs 跟随
- 政策方向:国家支持 / 中性 / 限制
- 产业链位置:上游(议价权强)/ 中游(议价权弱)/ 下游(品牌渠道)
- 进口替代 / 国产化率提升是 A 股常见 alpha 来源

**打分锚点**:
- 5 分:行业增速 >15%,集中度提升,政策支持,公司是龙头
- 3 分:行业稳定增长,公司中游位置
- 1 分:行业衰退或政策打压,公司竞争地位下滑

### 维度 6:时事热点 — "短期有没有催化剂或地雷"

**核心问题**:近期新闻、业绩预告、突发事件对短期股价的影响。

**关键来源**:
- 业绩预告 / 快报(akshare 有接口)
- 重大事项:股东减持、回购、定增、并购、股权激励
- 行业政策、监管处罚
- 突发事件:自然灾害、供应链事故、海外制裁

**打分锚点**:
- 5 分:近期有明确正面催化(业绩大超预期、大单落地、政策利好)
- 3 分:无重大消息
- 1 分:有明确负面事件(业绩暴雷、监管处罚、大股东减持)

**这个维度的时效性**:Claude 的训练数据有截止日期,近期热点可能不知道。脚本会尝试抓最近 30 天的公告,但如果数据缺失,**要明确告诉用户"Claude 没有实时新闻数据,建议用户自己核验近期公告"**,不要装作知道。

## 评级规则

六个维度打完分后,按以下规则给最终评级。**不是简单平均**,因为不同维度权重不同。

**权重分配**(经验值,可调):
- 公司财务 25%
- 估值 20%
- 技术面 15%
- 宏观 10%
- 行业 15%
- 时事 15%

加权得分 = Σ(维度分 × 权重),满分 5 分。

**评级映射**:
- ≥4.0 且无任何维度 ≤2:**强烈推荐**(多个维度共振 + 无明显短板)
- 3.5-4.0:**推荐**(基本面前景好,但有 1 个维度待观察)
- 2.5-3.5:**观望**(信号矛盾或方向不明)
- <2.5 或任一关键维度(财务 / 估值 / 行业)≤2:**回避**(有关键风险)

**关键否决项**:
- 财务维度 ≤2(生意本身有问题)→ 最高只能给"观望"
- 估值维度 =1(极度高估)→ 最高只能给"观望"
- 行业维度 ≤2(赛道明确下行)→ 最高只能给"观望"
- 时事维度 =1(有重大利空)→ 短期评级降到"观望"或"回避",即使其他维度都好

**为什么设置否决项**:这三个维度是"地基",地基坏了盖多高都白搭。设置否决项避免模型用"加权平均 3.8 推荐买"这种机械判断。

## 重要原则

1. **数据诚实**:脚本没抓到的数据,明确写"数据缺失",不要编。Claude 的训练数据可能过时,涉及"最新"的内容要提示用户核验。

2. **数字优先**:每个维度的判断都要先给数字,再给解读。避免"公司基本面良好"这种空话。

3. **交叉验证**:六维之间有冲突时要明确指出。例如"财务好但估值贵"或"基本面差但技术面好",这种矛盾恰恰是用户最需要的洞察。

4. **不确定性量化**:对每个维度的判断要给"信心度"(高/中/低)。低信心通常是数据不足或周期位置难判断。

5. **不做投资建议**:报告末尾加一句"本报告由 AI 生成,仅供参考,不构成投资建议。投资有风险,决策需谨慎。"

## 何时不用全套流程

如果用户只想要某个维度(例如"只看技术面"、"只算 PE 分位"),直接调对应脚本,不要跑全套六维。脚本之间相互独立,可以单独用。

## 何时升级

如果脚本某接口长期失败,看 `references/akshare_api.md` 找替代接口(akshare 经常变)。如果用户给的市场不在 A 股 / 港股(如美股、ETF、可转债),告诉用户当前 skill 不支持,问是否要扩展。

## 依赖

skill 运行需要以下 Python 库(`pip install` 一次性装好):

```
akshare matplotlib pandas markdown pypdf
```

- `akshare`:数据抓取(A股/港股/宏观)
- `matplotlib` + `pandas`:图表与数据处理
- `markdown`:Markdown → HTML
- `pypdf`:裁掉浏览器生成的 PDF 默认页眉页脚

还需系统装了 Edge(Windows 自带)或 Chrome,`generate_pdf.py` 用它 headless 渲染 PDF。
